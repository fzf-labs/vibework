## Why

任务详情页右侧目前只有预览容器，缺少可用的文件入口与更稳定的内容加载能力，导致用户难以发现/选择产物文件，且文本类文件经常显示为空。完善预览体验能让任务产出更容易被检查、复用与交付。

## What Changes

- 在任务详情页右侧“文件预览”中补齐文件列表与选择能力，仅展示当前工作目录/分支下的文件。
- 增强预览加载逻辑：文本/代码/Markdown/CSV 等可从文件路径读取内容，统一处理过大文件、加载失败与空内容状态。
- 增加必要的 UI 状态与操作入口（刷新列表、打开外部应用、复制路径/打开编辑器）。

## Capabilities

### New Capabilities
- `task-file-preview`: 在任务详情页右侧提供可用的文件列表与预览能力，聚焦工作目录文件的选择、加载与错误/空态处理。
